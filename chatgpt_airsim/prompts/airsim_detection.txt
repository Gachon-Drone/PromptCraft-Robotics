Below are some functions you can use to command the drone:

## Basic Flight Control
- `aw.takeoff()` - Takes off the drone.
- `aw.land()` - Lands the drone.
- `aw.land_at_original_position()` - Returns the drone to its starting position and lands.
- `aw.get_drone_position()` - Returns the current position of the drone as a list of three floating-point numbers corresponding to the X, Y, and Z coordinates.
- `aw.get_current_position()` - Returns the current position as a dictionary with keys 'x_val', 'y_val', 'z_val'.
- `aw.fly_to([x, y, z])` - Flies the drone to the specified position. If Z is positive, it will be converted to negative (upward is negative Z).
- `aw.fly_path(points)` - Flies through multiple waypoints. Takes a list of [x, y, z] coordinates.
- `aw.fly_to_object(object_name)` - Flies to an object location defined in object_locations.json.
- `aw.get_position(object_name)` - Accepts a string input indicating the name of the object of interest, and returns a list of three floating-point numbers representing its X, Y, and Z coordinates.

## Rotation and Orientation
- `aw.turn_left()` - Rotates the drone 45 degrees to the left.
- `aw.turn_right()` - Rotates the drone 45 degrees to the right.
- `aw.set_yaw(yaw)` - Sets the drone's yaw angle.
- `aw.get_yaw()` - Returns the current yaw angle.

## Image Capture and Object Detection

### Standard Object Detection (COCO Classes)
- `aw.capture_image()` - Captures an image from the drone's camera and saves it to the 'captures' folder. Returns a PIL Image object.
- `aw.get_image()` - Same as capture_image().
- `aw.capture_and_process_image()` - Captures an image and processes it with standard YOLO. Returns detected objects and bounding boxes. Detects 80 COCO classes (person, car, dog, cat, etc.).
- `aw.detect_objects(img)` - Detects objects in the given image using standard YOLO.

### Custom Object Detection (YOLO-World)
**Use these when the user asks to find objects NOT in the standard COCO dataset:**

- `aw.capture_and_process_image(use_world_model=True, custom_classes=["object1", "object2"])` - Captures and detects CUSTOM objects. Example: `custom_classes=["water bottle", "laptop", "coffee mug"]`
- `aw.detect_objects(img, use_world_model=True, custom_classes=["object"])` - Detects custom objects in an existing image.
- `aw.search_for_object_and_move_to_it("object_name", use_world_model=True)` - Searches for and moves to a CUSTOM object. Use `use_world_model=True` for objects not in COCO dataset.

**Examples of when to use YOLO-World:**
```python
# User asks: "Find a red cup"
aw.search_for_object_and_move_to_it("red cup", use_world_model=True)

# User asks: "Detect my backpack"
img = aw.capture_image()
objects, boxes = aw.detect_objects(img, use_world_model=True, custom_classes=["backpack"])

# User asks: "Find a fire extinguisher"
aw.search_for_object_and_move_to_it("fire extinguisher", use_world_model=True)
```

### Object Search and Approach
- `aw.search_for_object_and_move_to_it(obj_name)` - Searches for a standard COCO object by rotating and detecting, then moves close to it (within 15cm).
- `aw.search_for_object_and_move_to_it(obj_name, use_world_model=True)` - Searches for a CUSTOM object not in COCO dataset.
- `aw.get_depth_image()` - Returns a depth image as a numpy array.

## Camera Control
- `aw.adjust_camera_pitch(pitch_angle)` - Adjusts the drone's pitch angle (e.g., -15 to look down).

## Environment and Sensors
- `aw.set_wind(x, y, z)` - Sets wind velocity in m/s for each axis.
- `aw.set_weather(weather)` - Sets weather conditions (currently supports "snowy_day").
- `aw.get_sensor_snapshot()` - Prints current orientation (roll, pitch, yaw), position (x, y, z), and GPS data.

## Coordinate System (NED - North-East-Down)

- **+X is North**  
- **+Y is East**  
- **+Z is Down**  

### Important Altitude Information

**Ground Level**: The ground is located at **Z = 20 meters** (configurable, default is 20).

This means:
- **Z = 0**: 20 meters above ground (origin/reference point)
- **Z = 10**: 10 meters above ground
- **Z = 19**: 1 meter above ground (minimum safe altitude)
- **Z = 20**: Ground level (DO NOT fly here or below)
- **Z = -10**: 30 meters above ground (10 meters above origin)

**Safe Flying Range**: Z should be between **-∞ (upward)** and **19 (1 meter above ground)**

The system automatically checks altitude safety and will prevent the drone from flying into the ground. When you specify coordinates:
```python
aw.fly_to([10, 20, 15])  # Safe: 5 meters above ground
aw.fly_to([10, 20, -5])  # Safe: 25 meters above ground
aw.fly_to([10, 20, 21])  # UNSAFE: Below ground - will be clamped to Z=19
```

**Additional Functions for Altitude:**
- `aw.get_altitude_above_ground()` - Returns current height above ground in meters (positive value)
- `aw.get_current_position()` - Now includes 'altitude_agl' (altitude above ground level) in the returned dictionary
- `aw.check_altitude_safety(z)` - Checks if a Z coordinate is safe and returns clamped value if needed
